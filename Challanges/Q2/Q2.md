
## Coding Challenge: The Log File Analyzer

### **Problem Description**

You are a software engineer building a monitoring tool. Your system receives a log file as a list of strings. Each string represents an event containing a timestamp, a log level (`INFO`, `WARNING`, or `ERROR`), and a message, all separated by semicolons (`;`).

Your task is to write a function that processes these logs, counts the occurrences of each log level, and isolates the critical error messages.

---

### **Input Format**

* A **List** of strings, where each string follows the format: `"TIMESTAMP;LOG_LEVEL;MESSAGE"`
* A **String** representing a specific target log level you want to filter out (e.g., `"ERROR"`).

---

### **Constraints**

* If a log line does not contain exactly two semicolons (i.e., it doesn't split neatly into 3 parts), it is malformed. You must skip it.
* The `LOG_LEVEL` in the raw string might be in lowercase or uppercase (e.g., `"error"` or `"ERROR"`). Your function must handle this case-insensitively.
* The final message should have any leading or trailing whitespace removed.

---

### **Output Format**

Your function must return a **Dictionary** with exactly two keys:

1. `"counts"`: A nested **Dictionary** showing the total count of each log level (in all uppercase).
2. `"filtered_messages"`: A **List** of strings containing the cleaned messages that matched the target log level.

---

### **Sample Input**

```python
raw_logs = [
    "10:00:01;INFO; System started successfully ",
    "10:00:15;warning;Low memory detected",
    "10:01:00;ERROR;Database connection failed!",
    "10:01:05;INVALID_LINE_NO_SEMICOLONS", # Malformed log
    "10:02:20;error; Disk full, cannot write logs  "
]
target = "ERROR"

```

### **Sample Output**

```python
{
    "counts": {
        "INFO": 1,
        "WARNING": 1,
        "ERROR": 2
    },
    "filtered_messages": [
        "Database connection failed!",
        "Disk full, cannot write logs"
    ]
}

```

### **Explanation**

1. **String Splitting & Validation:** The code splits each string by `;`. It skips `"10:01:05;INVALID_LINE_NO_SEMICOLONS"` because it doesn't split into 3 distinct pieces.
2. **Case Insensitivity:** `"warning"` becomes `"WARNING"`, and `"error"` becomes `"ERROR"`.
3. **Dictionary Counting:** A dictionary keeps track of how many times each level appears.
4. **String Cleaning:** The message `" Disk full, cannot write logs  "` has its extra spaces stripped out using `.strip()`.

## Starter Code Template

Your friend can use this template to write and test their solution:

```python
def analyze_logs(logs: list, target_level: str) -> dict:
    # 1. Initialize your tracking data structures
    level_counts = {}       # Dictionary to store counts dynamically
    matching_messages = []   # List to store the target messages
    
    # Standardize target level to uppercase
    target_level = target_level.upper()

    # 2. Loop through each log entry
    for log in logs:
        # Split the string by the semicolon delimiter
        parts = log.split(";")
        
        # 3. Conditional validation: check if the line is malformed
        if len(parts) != 3:
            continue  # Skip corrupt data
            
        # Unpack the parts and clean them up
        timestamp = parts[0].strip()
        level = parts[1].strip().upper()  # Normalize to uppercase
        message = parts[2].strip()        # Remove trailing/leading spaces
        
        # 4. Update the counting dictionary
        if level in level_counts:
            level_counts[level] += 1
        else:
            level_counts[level] = 1
            
        # 5. Conditional check for the target level
        if level == target_level:
            matching_messages.append(message)

    # 6. Return the final structured dictionary
    return {
        "counts": level_counts,
        "filtered_messages": matching_messages
    }

# --- TEST CASE ---
raw_logs = [

    "10:02:20;error; Disk full, cannot write logs  "

result = analyze_logs(raw_logs, target)
print("Analysis Result:")
print(result)

```

* **Dynamic Dictionaries:** Tests the logic of *frequency counting* (checking if a key exists in a dictionary, adding it if it doesn't, or incrementing it if it does).
* **Data Structure Nesting:** Teaches them how to return a dictionary that contains both a nested dictionary and a list as values.
